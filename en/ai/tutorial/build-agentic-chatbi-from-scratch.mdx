## Step 1: Create Data Source Connections

The foundation of ChatBI is data, and the entry point is connecting to diverse data sources. XpertAI supports integration with mainstream databases and data warehouses, including but not limited to:

* MySQL / PostgreSQL / SQL Server
* SAP BW / SAP HANA
* ClickHouse / Doris / Hive / Presto
* Snowflake / Redshift
* Excel / CSV and other structured data sources

Simply configure the connection details in the **[Data Source Management](/docs/server/datasources/)** section, and the platform will automatically validate the connection.

---

## Step 2: Create a Semantic Model (Semantic Data Cube)

To help the agent understand the relationship between business concepts and data, we introduced the concept of a [**"semantic model"**](/docs/models/), which is a collection of data cubes. You can build data cubes through a graphical interface, defining:

* Dimensions (shared dimensions, inline dimensions)
* Measures (numeric fields, aggregation methods)
* Relationships between fact data tables and dimensions (primary-foreign key joins)

This step transforms the underlying data structure into an analysis model that is easy for AI models (and business users) to understand and use.

### Dimension Member Retrieval

The platform also provides the **[Dimension Member Retrieval](/docs/models/member-retriever)** function. This function dynamically synchronizes and vectorizes dimension member information, offering more accurate and efficient support for data analysis, enabling the agent to have stronger contextual awareness when identifying query intent and constructing filtering conditions.

---

## Step 3: Manage Semantic Model Metrics (Optional)

If you want to predefine commonly used KPI metrics (e.g., GMV, conversion rate, average order value) in the model, you can do so in the **[Metric Management](/docs/indicators/)** section for definition and categorization. These metrics will serve as key references for the agent to understand query intent and construct queries.

> âœ… **Tip**: This step is optional but highly recommended in complex business scenarios to improve the accuracy and business context understanding of intelligent Q&A. You can also implement equivalent logical calculations in the **[Calculated Members](/docs/models/mdx-modeling/calculated-member)** section.

---

## Step 4: Create ChatBI Data Model Configuration

This **[Conversational Model](/docs/ai/tool/chatbi/models)** configuration binds the semantic model to the agent's interaction context, serving as a bridge between the underlying data and the upper-layer ChatBI toolchain.

You need to:

* Select the created semantic model
* Choose the entities (data cubes) within the semantic model
* Provide a brief description and a detailed description to help the large language model understand its meaning

Once this step is complete, the agent can select and use the model based on its description.
