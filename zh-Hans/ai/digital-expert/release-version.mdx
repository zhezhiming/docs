Xpert 智能体支持用户方便地发布新版本，以满足持续优化和扩展需求。

- **版本迭代管理**：支持根据用户需求对现有 Agent 的功能或逻辑进行调整，并通过版本更新发布。
- **草稿与发布分离**：在发布新版本前，用户可以在草稿模式中测试和调整 Agent，确保稳定性和功能准确性。
- **版本发布日志**：记录每次发布的变更内容，方便追溯和理解版本更新的背景。

![Publish new version](/public/img/ai/publish.png)

## 版本管理功能

版本管理功能旨在确保 Xpert 智能体的版本演化透明可控，并帮助用户高效地管理多个版本的内容。

![Publish new version](/public/img/ai/xpert/publish-version.png)

- **版本历史查看**：用户可以查看 Agent 的所有历史版本，包括发布日期、版本号。
- **版本回滚**：在新版本出现问题时，可以快速切换回旧版本，保障业务的连续性和稳定性。
- **多环境支持**：支持在不同环境中管理多个版本，如测试环境、生产环境等。（即将推出）

## 发布到第三方平台

Xpert AI 平台支持将 Xpert Agent 发布到多个主流第三方平台，如飞书机器人、微信、钉钉等。通过集成这些平台，用户可以将智能助理和自动化任务扩展到日常的企业通信和工作流中，提升工作效率和用户体验。

Xpert AI 提供图形化配置界面，用户可以快速将智能体配置到目标平台，只需提供必要的 API 密钥和身份验证信息，系统会自动处理与各平台的连接。

![Publish to third-party platforms](/public/img/ai/publish-3-platforms.png)

Xpert Agent 会根据不同平台的特点进行优化，比如在微信和钉钉中支持丰富的图文消息、按钮交互等，而在飞书中则可以利用其更强的机器人功能进行集成和多样化交互。

### 发布到飞书机器人

对于发布到飞书，需要创建企业自建应用，并配置如下条件：

:::tip 前提条件
- **添加机器人能力**
- **将事件发送至 开发者服务器**
- 添加事件 `im.message.receive_v1`
- **将回调发送至 开发者服务器**
- 添加回掉 `card.action.trigger`
- 开通权限 `im:message.p2p_msg:readonly` `im:message.group_at_msg:readonly` `im:message` `im:message:send_as_bot`
:::

**开发者服务器** 使用发布中生成的 `Webhook url`。
