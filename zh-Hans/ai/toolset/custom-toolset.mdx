自定义工具集是用户根据自身需求创建和配置的工具集合，旨在扩展系统的功能和灵活性。通过自定义工具集，用户可以整合不同的服务和数据源，以满足特定的业务需求。

自定义工具集支持多种协议，其中包括：

- **OpenAPI 协议**：OpenAPI 是一种用于描述 RESTful API 的标准化格式。通过支持 OpenAPI 协议，自定义工具集可以轻松集成各种基于 REST 的服务，提供标准化的接口定义和交互方式。

- **OData 协议**：OData 是一种用于访问和查询数据的开放协议。支持 OData 协议的自定义工具集可以与各种数据源进行交互，提供灵活的数据查询和操作能力。

## 自定义 OpenAPI 工具集

自定义 OpenAPI 工具集允许用户通过提供 URL 或手动输入 Swagger OpenAPI 规范来创建工具集。系统会自动解析并识别规范中的 API 路径和参数，从而简化集成过程。
用户只需提供 OpenAPI 规范的 URL，或者直接在界面中输入规范内容，系统便会自动生成相应的接口定义，帮助用户快速构建和配置自定义工具集。

![Custom Toolset OpenAPI](/public/img/ai/custom-toolset-openapi.png)

## 自定义 OData 工具集

自定义 OData 工具集允许用户通过提供 OData 服务的 URL 或手动输入 OData 元数据内容来创建工具集。
系统会自动解析并识别元数据中的实体集（EntitySet），并生成相应的 CRUD（创建、读取、更新、删除）接口定义，从而简化集成过程。
用户只需提供 OData 服务的 URL，或者直接在界面中输入元数据内容，系统便会自动生成相应的接口定义，帮助用户快速构建和配置自定义工具集。


![Custom Toolset OpenAPI](/public/img/ai/custom-toolset-odata.png)

## 授权方法

自定义 API 工具集的授权方法包括以下几种：

1. **None**：无需任何授权即可访问 API。这种方法适用于公开的 API 接口，用户可以直接调用而无需提供任何凭证。

2. **Basic**：通过用户名和密码进行基本授权。用户需要在请求头中提供 Base64 编码的用户名和密码组合，以便进行身份验证。

3. **API Key**：使用 API 密钥进行授权。用户需要在请求中包含特定的 API 密钥，通常通过请求头或查询参数传递，以便验证用户的访问权限。

![Custom Toolset Authorization](/public/img/ai/custom-toolset-authorization-method.png)

这些授权方法为用户提供了灵活的选择，以满足不同 API 的安全需求。
