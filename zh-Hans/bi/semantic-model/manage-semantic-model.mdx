**语义模型管理界面** 向用户展示了有权限查看或者管理的语义模型, 用户可以查看, 创建或编辑语义模型.

语义模型列表以业务域为组织结构对语义模型进行树型展示.

点击界面右上角创建按钮可以新建不同类型的语义模型:

* **SQL** 语义模型
* **MDX** 语义模型
* **WASM** (SQL) 语义模型

## 创建向导

点击新建按钮, 在弹出的创建语义模型向导中选择已创建的 **数据源**, 点下一步选择 **服务目录**.

:::tip

**服务目录** 在一些数据库中对应用于隔离物理表的 **Schema**, 在另一些数据库中对应隔离物理表的数据库实例 **Database**, 在 XMLA 数据源中对应隔离模型的 **Catalog**.

:::

选择服务目录后进行下一步, 输入模型唯一名称和 *所属业务域*(可选), 选择是否使用 MDX 建模, 点击创建.

是否使用 MDX 建模:

* 是: 创建 MDX 模型
* 否: 创建 SQL 模型

## 管理语义模型

在语义模型列表中可以新增和删除模型, 点击语义模型名称转到语义模型编辑界面, 进行语义模型的维护和创建.

### 上传模型

点击上传按钮可以上传已保存下来的模型文件, 上传之前可以设置新模型的名称, 所属业务域和所使用的数据源.

## 权限

语义模型列表会列出以下有权限的模型：

- 用户创建的。
- 用户有权限的 [业务域](/docs/server/organization/business-area) 下的语义模型。

:::tip
管理员（ADMIN）角色对组织内所有业务域有权限。
:::
